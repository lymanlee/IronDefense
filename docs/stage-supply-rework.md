# 关卡制与宝箱补给改造设计稿

更新时间：2026-05-13

## 一、改造目标

本次改造要把当前“按波次暂停弹补给卡”的结构，升级为更有策略取舍的“关卡制 + 局内宝箱补给 + 局外武器成长”结构。

核心目标：

- 引入真正的关卡概念：`Chapter > Stage > Wave`
- 每个关卡由多轮波次组成，通关后进入下一关
- 武器升级改为全局成长，不再依赖单局补给临时进化
- 补给效果只在当前关卡内生效，进入下一关全部重置
- 补给触发方式改为战斗中击毁高血量宝箱后弹卡，形成“打敌人还是打宝箱”的即时取舍
- 保留当前爽感基础，不把游戏改成重度经营，而是提升中局决策密度

## 二、当前系统存在的问题

当前版本虽然已经有补给、武器进化、广告补拿等机制，但主要问题有：

- 补给触发时机固定：主要发生在波次暂停阶段，战斗过程中的决策较少
- 武器成长与局内补给耦合过深：武器分支更像“一局内临时拿到的强化”，长期目标感不够
- 波次是连续生存制，但缺少明确的“关卡完成”节点，阶段性成就感偏弱
- 补给池同时承担恢复、局内战术、结算资源、武器进化多个职责，系统语义不够清晰

因此，建议把系统拆成两条线：

- `局外长期成长`：武器解锁、升级、分支研究
- `局内临时策略`：宝箱补给、战术增益、应急修复、波次风险取舍

## 三、目标玩法闭环

新的核心循环建议如下：

1. 玩家在开始界面或车库中选择准备进入某个关卡
2. 进入关卡后，当前关卡从 `Wave 1` 开始推进
3. 战斗中持续刷敌人，同时按规则生成高血量补给宝箱
4. 玩家必须在“清敌保命”和“优先点掉宝箱拿补给”之间做取舍
5. 宝箱被击毁后立即暂停战斗并弹出补给卡牌，玩家从 3 选 1 中拿到局内增益
6. 当前关卡内的补给持续生效到本关结束
7. 关卡完成后结算金币、零件、武器经验等长期资源
8. 玩家回到局外武器系统，用累计资源强化主武器或解锁分支
9. 进入下一关时，局内补给全部清空，但武器长期成长保留

这个结构的重点是：

- `关卡内` 做短线策略
- `关卡外` 做长期养成

## 四、关卡结构设计

## 4.1 结构层级

建议增加如下内容结构：

- `Chapter`
  - 主题章节，如“桥头防线”“工业高架”“夜袭城区”
- `Stage`
  - 单个可进入关卡，如 `1-1`、`1-2`、`1-3`
- `Wave`
  - 关卡内的波次段落

首版不需要先做章节地图 UI，也可以先只做 `Stage 1 ~ Stage N` 的顺序解锁。

## 4.2 Stage 定义建议

每个关卡至少包含以下属性：

- `id`
- `name`
- `chapterId`
- `recommendedPower`
- `waveCount`
- `baseReward`
- `waveDefs`
- `chestRules`
- `enemyScale`
- `stageModifiers`

示例：

```ts
interface StageDef {
  id: string;
  name: string;
  chapterId: string;
  recommendedPower: number;
  waveCount: number;
  baseReward: {
    coins: number;
    parts: number;
    weaponExp: number;
  };
  waveDefs: StageWaveDef[];
  chestRules: StageChestRuleSet;
  enemyScale: {
    hpMult: number;
    atkMult: number;
    speedMult: number;
  };
  stageModifiers?: string[];
}
```

## 4.3 关卡完成条件

- 当前关卡全部波次完成
- 场上无残余敌人
- 若本关有 Boss，则 Boss 击杀后才允许结算

## 4.4 关卡失败条件

- 玩家武装车耐久归零
- 若已有复活系统，则先进入复活流程；复活失败后判定关卡失败

## 五、武器成长改造方向

## 5.1 武器成长改为全局永久成长

当前 `ExpSystem` 更接近“局内等级系统”。改造后建议拆成两层：

- `局内等级`
  - 仅决定本关战斗节奏，例如基础火力曲线、局内射击成长速度
- `局外武器成长`
  - 决定主武器等级、已解锁分支、长期强化路线

## 5.2 全局武器成长包含哪些内容

建议把永久成长拆成以下维度：

- 杀伤力：基础伤害、暴击、破甲
- 发射速度：射速、装填、连发密度
- 覆盖范围：爆炸半径、链弹范围、穿透深度
- 武器种类：机炮、狙击炮、榴弹炮、电磁链炮
- 武器分支：爆裂、穿透、电弧、燃烧等

## 5.3 经验值与金币关系

用户已经提出“武器升级可以通过经验值，或者把经验映射成金币来选择升级”。建议首版采用更稳的双货币结构：

- `关卡内掉落经验`
  - 用于本局即时升级节奏
- `关卡结算武器经验`
  - 用于提升该主武器的长期等级
- `金币`
  - 用于购买某些武器升级节点或解锁基础科技

建议不要直接把所有战斗经验 1:1 变金币，否则玩家会更倾向于纯堆经济，不利于战斗反馈。更好的方式是：

- 战斗中经验仍优先服务局内战斗节奏
- 关卡结算时，将本局总经验按比例转为 `weaponExp` 或少量金币奖励

例如：

- 本局累计经验 `800`
- 结算转化为 `weaponExp 320`
- 同时附带 `coins 80`

## 5.4 武器分支的归属

当前三条分支 `爆裂 / 穿透 / 电弧` 已经接入补给系统，但在新体系里建议改为：

- 不再作为局内普通补给出现
- 变成局外武器系统中的“永久解锁分支”
- 进入关卡前由玩家预设主分支，或在车库中装备默认分支

后续可以保留少量“局内临时偏转补给”，例如：

- 爆裂半径临时提升
- 电弧连锁数本关 +1
- 穿透层数本关 +1

这样局内补给不再决定“有没有这把武器”，而是决定“这把武器这关怎么打得更极端”。

## 六、宝箱补给系统设计

## 6.1 核心理念

宝箱不是白送奖励，而是战斗中的第二目标。

玩家会不断面对以下抉择：

- 现在优先清前排敌人，先保住护栏和血量
- 还是强行点掉宝箱，赌一个关键补给扭转局势

这会把补给行为从“暂停后的菜单选择”，变成“战斗中的高压资源争夺”。

## 6.2 宝箱触发规则

建议首版采用“按波次脚本 + 小概率动态补充”的混合方案。

每个关卡配置每波最多出现几个宝箱，并指定生成窗口：

- `Wave 1-2`：最多 1 个宝箱，用于教学和建立认知
- `Wave 3-4`：1 到 2 个宝箱
- `Boss Wave`：固定 1 个稀有宝箱或生存宝箱

生成时机建议：

- 波次开始后 `8~15` 秒
- 或当前波次敌人击杀率达到某阈值后
- 或玩家血量低于某阈值时提高生存类宝箱概率

## 6.3 宝箱存在规则

建议首版采用以下约束：

- 同一时刻场上最多存在 `1` 个宝箱
- 宝箱生成在桥面上半区或中段，不直接刷在车旁
- 宝箱有持续时间，超时未击毁则撤离或自爆消失
- 宝箱不会主动攻击，但血量较高，需要玩家投入一定输出

这样能明确制造决策压力，而不是无脑顺手打掉。

## 6.4 宝箱血量与风险

建议宝箱血量按当前关卡波次强度缩放：

```ts
chestHp = currentWaveBaseEnemyHp * chestHpMultiplier
```

首版建议：

- 普通宝箱：`8 ~ 12` 倍普通敌人血量
- 精英宝箱：`14 ~ 20` 倍普通敌人血量
- 稀有宝箱：`Boss 波次专属`

还可以加上“打箱代价”：

- 宝箱被命中时会短时间吸引火力注意力，但不移动
- 某些宝箱周围会附带少量护卫敌人
- 宝箱超时后失效，形成真实机会成本

## 6.5 宝箱类别

建议首版只做 5 类，方便玩家形成记忆：

- `火力箱`
  - 倾向掉落伤害、射速、多重弹、范围强化
- `生存箱`
  - 倾向掉落回血、护盾、减伤、临时装甲
- `控制箱`
  - 倾向掉落冻结、减速、击退、电磁干扰
- `资源箱`
  - 倾向掉落本关金币倍率、额外零件、结算加成
- `稀有箱`
  - 倾向掉落高品质复合补给，通常在高波次或 Boss 波触发

## 6.6 宝箱击毁后的卡牌逻辑

宝箱被击毁后建议：

1. 游戏进入短暂停顿
2. 弹出对应宝箱类型的补给卡牌面板
3. 默认提供 `3 选 1`
4. 玩家选择完成后立即恢复战斗

卡池不再是“所有补给混抽”，而是“按箱子类别定向抽取，再混入少量泛用项”。

例如：

- 火力箱：
  - 高爆弹药
  - 快装弹链
  - 过载供弹
  - 爆裂半径强化
  - 穿透层数 +1
- 生存箱：
  - 战地包扎
  - 强化底盘
  - 临时装甲
  - 护栏紧急维修
- 控制箱：
  - 冷冻地雷
  - 冲击弹头
  - 粘滞燃烧剂
  - 电弧麻痹

## 6.7 补给生效范围

所有宝箱补给均为 `stage-local`，即：

- 仅在当前关卡内生效
- 进入下一关时全部清空
- 不写入永久存档

建议把局内效果按持续方式分成三类：

- 即时类：
  - 回血
  - 立即发放少量金币
- 本关持续类：
  - 伤害倍率
  - 射速倍率
  - 减伤
  - 控制强化
- 下一波触发类：
  - 开场空袭
  - 首批冻结

改造后，“本关持续类”会比“下一波触发类”更重要，因为补给触发发生在波内，不再完全绑定波间休息。

## 七、补给与关卡的配合规则

## 7.1 每关的补给节奏

建议每关控制在 `2~4` 次有效补给选择。

原因：

- 太少会让宝箱系统存在感不够
- 太多会拖慢战斗节奏，变成频繁打断

推荐区间：

- 短关卡：`2` 次
- 标准关卡：`3` 次
- Boss 关卡：`3~4` 次，其中最后一次偏高品质

## 7.2 品质递进

同一关内可以设计补给品质递进：

- 第一箱：基础战术补给
- 第二箱：定向强化
- 第三箱：稀有强化或复合强化

这样玩家会更愿意撑到后期，而不是首箱拿完就觉得本关策略已经结束。

## 7.3 失败保护

为了避免纯随机把局玩崩，建议增加弱保底逻辑：

- 玩家血量过低时，生存箱权重提高
- 连续两次未打掉宝箱时，下一次宝箱血量略降
- Boss 波前如果本关一次补给都没拿到，保底刷一个低风险宝箱

## 八、对当前代码结构的改造建议

## 8.1 建议新增的核心模块

- `StageManager.ts`
  - 负责关卡生命周期、关卡波次推进、关卡结算
- `StageRuntimeState.ts`
  - 记录本关局内补给状态、临时加成、宝箱生成状态
- `SupplyChest.ts`
  - 管理宝箱血量、受击、销毁、超时、掉卡触发
- `WeaponProgressionManager.ts`
  - 管理武器长期等级、已解锁分支、分支装配
- `StageConfig.ts`
  - 或先继续挂在 `GameConfig.ts` 中，后续再拆

## 8.2 当前代码可复用部分

以下逻辑可以复用，不需要推倒重来：

- `GameManager._showSupplyPanel()`
  - 可继续承担“弹卡 UI”职责
- `GameManager._pickSupplyOptions()`
  - 可重构为“按宝箱类型抽卡”
- `GameManager._applySupplyOption()`
  - 可保留为局内效果应用中心
- `ProgressManager`
  - 可继续管理金币、零件、永久成长
- `ExpSystem`
  - 可保留局内经验功能，但后续需要与全局武器成长职责拆分

## 8.3 当前代码需要弱化或移除的部分

以下逻辑不再适合作为主流程：

- `_maybeShowSupplyOffer()`
- `_shouldOfferSupply()`
- `gameplay.supply.offerEveryWaves`
- `gameplay.supply.bossWaveEvery`
- “波次暂停即弹补给”的主逻辑

这些内容不一定立即删除，但应逐步退场，先做兼容开关更稳。

## 8.4 推荐的兼容改造顺序

第一步不要直接砍老系统，建议先做双模式兼容：

- `supplyMode: 'wave_break' | 'chest_trigger'`

这样可以：

- 先保留旧系统用于回归验证
- 新系统出问题时可以快速切回
- 方便做 A/B 手感对比

## 九、数据配置建议

建议在配置层新增：

```ts
stageSystem: {
  enabled: true,
  defaultSupplyMode: 'chest_trigger',
  chest: {
    maxActiveCount: 1,
    popupPauseScale: 0,
    popupDuration: 999,
    despawnSeconds: 14,
    hpMultiplierByQuality: {
      normal: 8,
      elite: 12,
      rare: 18,
    },
  },
  stageSupply: {
    defaultChoiceCount: 3,
    maxSelectionsPerStage: 4,
    pityRules: {
      lowHpSurvivalBoost: true,
      missedChestDropCompensation: true,
    },
  },
}
```

以及每关独立配置：

```ts
stages: [
  {
    id: 'stage_1_1',
    name: '桥头试探',
    waveCount: 4,
    chestRules: {
      guaranteedChests: [
        { wave: 2, type: 'survival', quality: 'normal' },
        { wave: 4, type: 'firepower', quality: 'elite' },
      ],
    },
  },
]
```

## 十、UI 与编辑器改造建议

这部分后续进入实现时，优先通过 Cocos Creator `3.8.8` 的 MCP 操作完成。

需要新增或调整的内容大致包括：

- 主界面增加关卡入口与当前关卡信息
- 车库界面增加武器成长页签或分支装备区
- 战斗 HUD 增加：
  - 当前关卡编号
  - 当前波次
  - 本关已拿补给数
  - 宝箱来袭提示
- 场景中新增宝箱节点或宝箱预制体
- 宝箱击毁后复用现有补给弹窗，必要时再优化样式

后续实际落地时，以下操作优先走 MCP：

- 创建场景节点
- 挂载组件脚本
- 绑定按钮点击事件
- 调整 HUD 节点层级
- 创建/实例化预制体

## 十一、分阶段实现计划

## Phase 1：把补给触发从“波间”改成“宝箱击毁”

目标：

- 保留现有波次系统
- 新增宝箱对象
- 宝箱被击毁后触发补给卡
- 先不做完整关卡制

具体工作：

- 新增 `SupplyChest.ts`
- 在 `GameManager` 中接入宝箱生成与销毁回调
- 重构 `_pickSupplyOptions()` 支持按箱子类型抽卡
- 保留 `_applySupplyOption()` 作为效果中心
- 用配置开关兼容旧的波间补给

这样能最快验证“战斗中抢箱子”的手感是否成立。

## Phase 2：引入关卡制

目标：

- 从“无限波次生存”过渡到“单关多波次通关”
- 完成本关结算与下一关入口

具体工作：

- 新增 `StageManager`
- 引入 `StageDef`
- 把当前 `waveDefs` 改造成“按关卡组织的 waveDefs”
- 在结算界面增加“关卡完成 / 关卡失败”语义

## Phase 3：把武器分支改成局外永久成长

目标：

- 三条分支从补给池中剥离
- 改为车库内永久解锁、选择与升级

具体工作：

- 新增武器成长存档结构
- `ProgressManager` 扩展武器成长数据
- `GarageScreen` 增加武器页签或武器图鉴页
- 战斗开始时由已装备分支初始化 `PlayerCar / Bullet`

## Phase 4：做补给和武器的深度联动

目标：

- 同一把武器在不同关卡内能打出不同临时流派

例如：

- 爆裂机炮更容易刷出范围强化
- 电弧机炮更容易刷出控制强化
- 穿透机炮更容易刷出破甲或直线清场强化

这样既保留永久成长，也保留局内随机策略。

## 十二、建议优先级

结合当前项目状态，建议执行顺序如下：

1. 先完成 `Phase 1`，验证宝箱补给手感
2. 再做 `Phase 2`，补齐关卡结构
3. 再推进 `Phase 3`，把武器分支彻底搬到局外
4. 最后做 `Phase 4` 的联动强化

原因是：

- 宝箱补给是当前玩法差异化最明显的改动
- 关卡制是中层框架，需要在宝箱手感确认后再固化
- 武器局外化改动范围更大，适合在关卡循环确定后接入

## 十三、与当前武器升级待办的衔接

当前 `docs/weapon-upgrade-todo.md` 中提到的三条分支平衡验证仍然有价值，但要注意目标要切换：

- 短期：验证 `爆裂 / 穿透 / 电弧` 作为武器分支本身是否成立
- 中期：把它们从“补给进化”迁移成“永久分支”
- 长期：再围绕不同分支设计定向补给偏好

也就是说，武器分支这部分不需要推翻，重点是“换归属、换入口、换成长层级”。

## 十四、下一步建议

文档确定后，下一步建议直接进入 `Phase 1` 的技术拆解，先做一版最小可运行方案：

- 新增宝箱敌对目标
- 战斗中生成宝箱
- 击毁后弹补给卡
- 局内补给效果沿用现有系统
- 保留旧补给模式作为开关

这样我们可以先把最关键的玩法差异跑起来，再继续接关卡制和武器局外化。
