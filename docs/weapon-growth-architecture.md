# 武器成长体系重构设计稿

更新时间：2026-05-17

## 一、文档目标

这份文档用于沉淀当前项目的武器成长底层设计，解决“名义上有等级、实际上主要靠补给成长”的结构混乱问题。

文档目标有三点：

- 明确当前版本武器成长系统的真实状态
- 给出后续可持续迭代的目标结构
- 约束配置项与代码职责，避免后续继续出现语义混用

这份文档不是单纯的玩法提案，而是后续实现、调参、验收都要持续对照的系统设计基线。

## 二、当前版本现状

截至当前版本，项目已经具备以下特征：

- 关卡内主要成长来源已经切换为宝箱补给卡
- 武器分支 `爆裂 / 穿透 / 电弧` 已可在局内获取并形成明显差异
- 补给卡可在单关内持续叠加，进入下一关后清空
- HUD 已经不再展示局内经验，改为展示本关击杀进度

但武器底层曾经保留过一套历史遗留结构：

- `ExpSystem` 过去以“经验 / 等级系统”命名
- `weaponBase` 已承担基础武器模板职责
- 旧的 `expLevels` 历史阈值表已经移除
- 局内经验升级链路已停用，波次与敌人已不再承载经验掉落语义

当前意味着：

- “基础武器档位系统”已经正式独立
- “补给系统”已经成为实际主成长系统
- 代码主结构已不再依赖局内经验链路

## 三、当前真实火力关系

当前版本的武器强度由四层叠加决定：

### 3.1 局外永久成长

来源：

- `ProgressManager`

当前已生效内容：

- 武装车最大生命
- 基础伤害倍率
- 开局基础武器档位

特点：

- 跨关卡永久保留
- 偏账号长期成长

### 3.2 基础武器档位

来源：

- `weaponBase`
- `WeaponTierSystem`

当前承担内容：

- 基础伤害
- 基础射速
- 基础子弹速度
- 基础并发数
- 基础连发数
- 基础夹角

特点：

- 本质上已经更像“基础武器模板”
- 不是当前主玩法中的局内动态成长来源

### 3.3 局内补给构筑

来源：

- 宝箱补给卡

当前已生效内容：

- 伤害倍率
- 射速倍率
- 连发增加
- 并发增加
- 穿透增加
- 电弧增加
- 电弧范围增加
- 爆裂范围增加
- 击退 / 减速 / 护盾等战术效果

特点：

- 当前单关内最核心的成长来源
- 直接决定玩家的爽感与构筑方向

### 3.4 武器分支进化

来源：

- 武器分支补给

当前分支：

- 原版
- 爆裂
- 穿透
- 电弧

特点：

- 决定单关内的流派方向
- 决定子弹行为、特效与后续可出现的强化补给

## 四、当前系统的核心问题

### 4.1 名称与职责不一致

旧 `ExpSystem` 命名曾经会让后续开发者自然认为：

- 它管理局内经验升级
- 它是单局战斗成长主轴

但实际上它目前更接近：

- 基础武器档位读取器
- 调试起始档位配置入口

### 4.2 成长主轴已经切换，但配置语义没有完全切换

在完成本轮重构前，玩家真正能感知到的成长主要来自补给卡，而配置和代码中曾残留：

- 历史经验阈值表
- “等级解锁分支”这类旧设计语义

这会带来三个问题：

- 调参与排障成本高
- 新功能接入时容易接错层级
- 文档与实现容易持续背离

### 4.3 基础档位与局内补给的边界不清

目前仍然容易出现类似问题：

- 这个数值该调 `weaponBase` 还是调补给卡
- 并发数增加后该继承哪一档基础夹角
- 分支增强是长期内容还是本局构筑内容

如果不先明确底层结构，只继续调数值，会越来越难维护。

## 五、目标结构

后续武器成长体系统一拆成三层。

### 5.1 第一层：局外武器科技成长

定义：

- 玩家跨关卡保留的长期武器成长

作用：

- 决定开局基础武器档位
- 决定是否解锁更高阶基础机炮模板
- 后续可扩展为更多主武器家族的解锁前提

特点：

- 永久保留
- 是中长期目标

建议命名：

- `WeaponTech`
- `WeaponTier`
- `WeaponProgression`

不建议继续称为“局内等级”。

### 5.2 第二层：局内补给构筑

定义：

- 玩家在单个关卡内，通过宝箱补给不断叠加出来的即时火力结构

作用：

- 决定本关火力膨胀速度
- 决定玩家当前关卡的爽感与策略取舍

特点：

- 只在当前关卡有效
- 进入下一关后全部清空

### 5.3 第三层：局内武器分支

定义：

- 玩家在本关中选择并强化的武器风格

当前分支：

- 爆裂
- 穿透
- 电弧

作用：

- 提供战斗风格差异
- 为补给池提供构筑方向

特点：

- 本关内生效
- 与补给池联动

## 六、系统职责定义

### 6.1 WeaponTierSystem

当前主类已切换为 `WeaponTierSystem`。

它只负责：

- 读取当前基础武器档位
- 根据档位返回基础武器模板
- 提供调试时的起始武器档位设置

它不再负责：

- 局内击杀经验累积
- 单局自动升级
- 敌人经验掉落处理

### 6.2 PlayerCar

`PlayerCar` 只负责把“基础模板 + 本关构筑结果”组合成最终开火行为。

它应承担：

- 根据基础模板决定初始开火方式
- 根据补给叠加决定最终并发 / 连发 / 射速
- 根据分支决定表现与行为差异

它不应承担：

- 长期成长存档逻辑
- 补给系统决策逻辑

### 6.3 GameManager

`GameManager` 只负责当前关卡内的构筑状态。

它应承担：

- 当前关卡内的补给状态
- 当前主补给池已聚焦为“火力 / 控制 / 分支强化”，生存类与资源运营类补给已从宝箱三选一中移除
- 当前宝箱语义同步收敛为“火力箱 / 控制箱 / 稀有箱”，前期优先起手火力，中期提高分支出现率，后期偏向分支专精强化
- 当前武器分支状态
- 当前关卡的临时火力增益状态

它不应承担：

- 基础武器档位配置来源定义
- 永久成长存档逻辑

### 6.4 ProgressManager

`ProgressManager` 只负责局外长期成长。

它应承担：

- 金币 / 零件 / 后续全局经验存档
- 永久升级项
- 武器科技档位或武器研发等级

## 七、配置规则约束

这一节用于给后续配置改造定边界，避免重新混乱。

### 7.1 `weaponBase`

定位：

- 基础武器模板表

职责：

- 描述不同基础武器档位的开局底座

当前保留字段：

- `profileNames`
- `baseSpreadCount`
- `baseBurstCount`
- `spreadAngle`
- `burstSpeedScales`
- `speed`
- `damage`
- `fireRate`

后续约束：

- 这里只放“基础模板”
- 不放局内补给增益
- 不放分支增强值

### 7.2 `bullet`

定位：

- 子弹碰撞 / 表现配置

建议只保留：

- `radius`
- 后续可能增加的纯表现参数

不再承担：

- 按档位成长的伤害 / 速度 / 射速

### 7.3 `waves`

定位：

- fallback / debug 波次模板

当前主玩法优先使用：

- `waveDefs`
- `stages`

约束：

- 不再把 `waves` 视为主内容编排入口
- 可保留给调试与越界回退

### 7.4 全局武器档位

当前状态：

- 历史 `expLevels` 表已删除
- 基础武器档位上限直接读取 `weaponBase` 档位数量
- 开局基础武器档位由 `ProgressManager` 的永久升级 `weapon_tier` 决定

后续处理建议：

- 若未来接入“全局武器科技经验”，应作为独立长期系统新增
- 不再回退到局内经验表驱动基础武器档位

## 八、推荐的新数值关系

后续统一使用如下总公式理解武器强度：

`最终火力 = 基础武器档位 × 局外永久成长 × 局内补给构筑 × 分支修正`

对应解释：

- 基础武器档位：决定开局底座
- 局外永久成长：决定长期强度抬升
- 局内补给构筑：决定本关火力膨胀速度
- 分支修正：决定战斗风格与特效行为

这套关系要求：

- 长期成长负责“开局更强”
- 局内补给负责“这一关越打越爽”
- 分支负责“这一关打得不一样”

## 九、后续实现顺序

### 9.1 第一阶段：语义与文档收口

目标：

- 先让概念正确

任务：

- 将 `ExpSystem` 的职责文档化为“基础武器档位系统”
- 调试面板中的“等级”改名为“武器档位”
- 为现有配置增加清晰注释

### 9.2 第二阶段：切断局内经验残留

目标：

- 明确局内已不走经验升级

任务：

- 清理 `addExp()` 相关残留语义
- 清理波次与敌人上的经验残留字段
- 避免后续新逻辑误接局内经验

### 9.3 第三阶段：接入全局武器科技成长

目标：

- 让基础武器档位真正有局外来源

任务：

- `ProgressManager` 增加武器科技等级或全局经验
- 开局根据科技等级映射到 `weaponBase` 档位

### 9.4 第四阶段：补给池进一步体系化

目标：

- 让局内构筑更稳定、更可控

任务：

- 按标签和流派重构补给卡池
- 将分支增强与通用火力补给分层
- 调整宝箱品质与补给强度对应关系

## 十、文案与沟通规则

为了避免团队在沟通中继续混淆，后续建议统一使用以下说法：

- 不说“局内等级”，优先说“基础武器档位”
- 不说“经验升级武器”，优先说“局外武器科技成长”
- 不说“临时升级武器等级”，优先说“本关补给构筑”
- 不说“拿到武器分支就永久拥有”，优先说“本关分支路线”

## 十一、待办清单

### 高优先级

- 让 `WeaponTierSystem` 接入真正的局外武器科技来源
- 调试面板与配置继续统一使用“武器档位”语义
- 为 `GameConfig` 增补成长体系配置注释

### 已完成的历史清理

- `ExpSystem` 兼容壳已删除
- 主流程已切换到 `WeaponTierSystem`
- `WaveData.exp` / `waves.exp` / `waveScaling.expAdd` 已移除
- 敌人经验掉落已退出主结构

### 中优先级

- 设计“全局武器科技等级”存档结构
- 打通 `ProgressManager -> WeaponTierSystem -> GameManager`
- 将补给卡按“通用 / 分支 / 运营 / 生存”重新分类

### 低优先级

- 继续扩展更多主武器家族
- 扩展更多分支流派
- 引入更细的构筑权重控制

## 十二、结论

当前版本最需要的不是继续局部调数值，而是先把成长体系的底层概念理顺。

明确结论如下：

- 局内经验成长已经不是主轴
- 宝箱补给构筑已经是当前主成长来源
- `weaponBase` 应正式解释为基础武器模板表
- 后续应把“等级”收编为“局外武器科技档位”

后续所有武器相关改造，都应以这份文档作为长期对照基线。
